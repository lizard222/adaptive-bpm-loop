# Спайк T-28: SpiffWorkflow — граничные таймеры и эскалации

**Дата:** 09.07.2026 · **Версия движка:** SpiffWorkflow 3.1.2 · **Результат прогона:** 13/13 проверок (`run_spike.py`)

## Вопрос спайка (риск из thesis_spec_adaptive_bpm.md §10)

Потянет ли SpiffWorkflow конструкции BPMN, критичные для процесса «Защита ВКР»:
граничные таймерные события (напоминания, эскалации) — или таймеры придётся выносить Контролеру?

## Вердикт: ✅ движок тянет, запасной план не нужен

Проверено на фрагменте «Подготовить приказ о допуске» с двумя граничными таймерами:

1. **Non-interrupting таймер (напоминание)** — срабатывает в срок, порождает ветку напоминания, исходная задача остаётся активной. ✅
2. **Interrupting таймер (эскалация)** — срабатывает в срок, отменяет исходную задачу (CANCELLED), процесс корректно завершается по ветке эскалации. ✅
3. Если исполнитель успевает выполнить задачу — таймеры корректно гасятся, ложных срабатываний нет. ✅

## Находки для этапа реализации (T-31, T-34)

1. **Windows + кириллица:** `parser.add_bpmn_file()` читает файл в системной кодировке (cp1251) и падает на кириллических названиях задач. Загружать через lxml: `parser.add_bpmn_xml(etree.parse(path), filename=path)`.
2. **Таймеры пассивны:** движок проверяет их только при вызове `refresh_waiting_tasks()`. Оркестратору (T-31) нужен периодический цикл «refresh → do_engine_steps» (или проверка при каждом обращении к экземпляру). Частота тика = точность срабатывания таймеров.
3. **Изоляция данных веток:** переменные, записанные в ветке non-interrupting события (`reminder_fired`), НЕ сливаются в общие данные процесса — это корректная семантика BPMN. Вывод для архитектуры: факты «напоминание отправлено», «эскалация выполнена» фиксируются в `event_log` (ФТ-С-5), а не в переменных процесса. Подтверждает принятое решение.
4. **Выражение таймера — Python-выражение:** в `timeDuration` пишется `"PT2S"` (строка в кавычках), т.к. SpiffWorkflow вычисляет его как Python. Это позволит брать длительности из переменных процесса — удобно для корректировок контура адаптации (нормативные сроки как параметры, ФТ-С-6.3).
